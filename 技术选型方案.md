## 二、技术栈详情

### 2.1 后端技术栈

| 技术 | 版本 | 用途 | 选型理由 |
|------|------|------|----------|
| **Node.js** | ≥18 LTS | 运行时 | 全栈统一语言 |
| **Express** | latest | Web框架 | 轻量灵活，生态成熟 |
| **TypeScript** | ≥5.0 | 开发语言 | 类型安全，IDE支持好 |
| **Sequelize** | latest | ORM | MySQL兼容好 |
| **MySQL** | ≥8.0 | 数据库 | 用户确认 |

**备选**：
- NestJS：如果需要更结构化的后端架构
- Prisma：如果是PostgreSQL

### 2.2 前端技术栈

| 技术 | 版本 | 用途 | 选型理由 |
|------|------|------|----------|
| **Vue 3** | ≥3.3 | UI框架 | 用户确认 |
| **TypeScript** | ≥5.0 | 开发语言 | 类型安全 |
| **Vite** | ≥5.0 | 构建工具 | 速度快 |
| **Element Plus** | latest | UI组件库 | Vue生态完善 |
| **Pinia** | latest | 状态管理 | Vue官方推荐 |
| **ECharts** | ≥5.0 | 图表库 | 图表类型全 |
| **Axios** | latest | HTTP请求 | 通用 |

### 2.3 大模型配置

| 配置项 | 说明 |
|--------|------|
| **base_url** | 用户后续提供 |
| **api_key** | 用户后续提供 |
| **模型名称** | 用户后续确认 |

**调用方式**：OpenAI兼容API格式

```typescript
// 大模型调用示例
const response = await fetch(`${base_url}/v1/chat/completions`, {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'Authorization': `Bearer ${api_key}`
  },
  body: JSON.stringify({
    model: '模型名称',
    messages: [{ role: 'user', content: '用户问题' }],
    temperature: 0.7
  })
});
```

### 2.4 数据库设计

**数据库：MySQL 8.0**

| 表分类 | 说明 |
|--------|------|

| **系统配置表** | 系统配置项 |

## 五、大模型配置说明

> 用户确认使用本地部署的大模型，以下是配置占位符

在 `backend/src/config/` 中创建 `llm.config.ts`：

```typescript
export const llmConfig = {
  base_url: '用户提供后填入',  // 例如: http://localhost:8000
  api_key: '用户提供后填入',
  model: '用户确认后填入',    // 例如: qwen-chat, chatglm3 等
  temperature: 0.7,
  max_tokens: 2000
};
```
