## 🎯 优化计划

### Phase 4: 查询能力增强 (优先级: 高)

#### 4.1 多条件筛选
**目标**: 支持 WHERE 多条件组合查询

**示例查询**:
- "华东地区零售客户的销售额"
- "2024年已完成的订单数"

**任务**:
- [ ] NLU Agent 识别多条件
- [ ] SQL Generator 支持多 WHERE 条件
- [ ] 测试验证

#### 4.2 时间范围查询
**目标**: 支持相对时间和绝对时间查询

**示例查询**:
- "最近7天的销售额"
- "2024年第一季度的订单数"
- "本月客户增长"

**任务**:
- [ ] NLU Agent 识别时间表达式
- [ ] SQL Generator 支持时间函数
- [ ] 测试验证

#### 4.3 排序功能
**目标**: 支持 ORDER BY 排序

**示例查询**:
- "销售额最高的前10个城市"
- "订单数最少的客户"

**任务**:
- [ ] NLU Agent 识别排序意图
- [ ] SQL Generator 支持 ORDER BY
- [ ] 测试验证

---

### Phase 5: 多轮对话 (优先级: 中)

**目标**: 支持追问和上下文关联

**示例对话**:
```
用户: 销售额是多少
系统: 销售总额为 7.15 亿元
用户: 按客户类型分组呢？
系统: [按客户类型分组的结果]
用户: 其中零售客户呢？
系统: [零售客户的详细数据]
```

**任务**:
- [ ] 实现会话上下文管理
- [ ] NLU Agent 支持追问识别
- [ ] 上下文传递给 SQL Generator

---

### Phase 6: 澄清问题交互 (优先级: 中)

**目标**: 当理解不确定时，主动询问用户

**示例**:
```
用户: 销售额
系统: 您想查询哪种销售额？
       1. 总销售额
       2. 某地区的销售额
       3. 某产品的销售额
```

**任务**:
- [ ] Clarification Agent 完善
- [ ] 前端支持选项选择
- [ ] 多轮对话流程
